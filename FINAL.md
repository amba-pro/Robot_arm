Цели финальной работы
Выполнение финальной работы позволит вам:
● Закрепить навыки работы с bash-скриптами и OC Linux.
● Закрепить навыки работы с Git.
● Закрепить навыки работы с контейнерами Docker.
● Продемонстрировать навыки работы с ROS для разработки ПО роботов.
Что нужно сделать
Представьте, что вы создаёте систему управления роботом. Вам предстоит описать
его архитектуру и функции, а также обеспечить поддержку датчиков. Для этого
необходимо разработать инструкцию по установке, настройке и использованию
(bash-скрипты, Docker). Чтобы справиться с заданием, подготовьте приложение для
сбора и обработки данных с датчиков робота в ROS 1 / ROS 2 (далее ROS).
Требования
● Придумайте концепцию робота: например, робот-пылесос.
● Составьте схему подключения конкретных датчиков и исполнительных
устройств к вычислителю робота на базе OC Linux. Например: LIDAR Velodyne
VLP-16 подключается по 1Gbit Ethernet, контроллеры двигателей колёс
ODrive — по CAN, IMU-датчик XSens MTI-1 — по SPI и так далее.
● Определите библиотеки и пакеты ROS, которые необходимы для работы
датчиков в OC Linux и ROS. Создайте скрипт, который установит все библиотеки
и пакеты.
● Добавьте скрипт, который настроит удалённый доступ по SSH.
● Создайте репозиторий Git с ROS Workspace, в котором будет README.md
с инструкцией по установке, запуску и другими важными действиями. Выведите
в README.md структурную схему.
● Создайте ROS-пакет с LAUNCH-файлом, чтобы запустить ROS-узлы для
работы с датчиками и исполнительными устройствами. Добавьте в
LAUNCH-файл запуск по параметру rviz или rqt (rviz:=true, rqt:=true).
● Подключите и проверьте работу одного из датчиков или исполнительных
устройств. Это может быть встроенная веб-камера ноутбука, геймпад или
джойстик, заряд батареи, мышка и клавиатура. Визуализируйте данные с этого
устройства в rviz или rqt.
● Создайте Dockerfile для сборки docker-образа со всеми необходимыми
зависимостями (библиотеками и пакетами ROS, доступом по SSH).
Советы и рекомендации
● Цель финальной работы не в создании системы управления роботом,
а в подготовке всего необходимого для этого процесса.
● Возьмите за основу Git-репозиторий, который вы разработали в практическом
задании из модуля 4.
● Вы можете придумать любой набор датчиков и исполнительных устройств
(приводов), главное, чтобы у них была поддержка ROS. Опираясь на них,
сформулируйте требования ко всей системе.
● Не нужно покупать и подключать все датчики и исполнительные устройства,
которые вы придумали. Важно только подготовить всё для их подключения
на основе рекомендаций производителя и разработчиков пакетов ROS.
Конечно, чаще всего проблемы с оборудованием возникают именно в процессе
подключения. Они могут быть очень специфичны, поэтому их решение
не входит в этот курс. Однако мы надеемся, что наши материалы подскажут
вам, в каком направлении искать решение.
● Выполнять работу удобно в VS Code с расширениями для Bash, ROS, Docker
и Remote Containers (для подключения VS Code к docker-контейнеру).
Что оценивается
● Корректность выполнения скриптов по установке необходимых библиотек
и пакетов ROS для работы датчиков.
● Корректность сборки docker-образа и запуска docker-контейнера.
● Корректность выполнения всех условий задания.
